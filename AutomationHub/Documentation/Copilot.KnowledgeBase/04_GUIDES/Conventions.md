# Convenções de Código, Commits e Branches

## Código C#
- `nullable` habilitado, uso de `var` quando claro
- Comentários XML para APIs públicas
- Controllers pequenos, lógica em Services

## Commits
- Use Conventional Commits:
  - `feat: ...`, `fix: ...`, `docs: ...`, `refactor: ...`, `test: ...`, `chore: ...`

## Branching
- `main`: estável
- `feature/*`, `fix/*`, `docs/*`: trabalho em andamento
- Pull Requests com revisão e checagens (build, testes, análise)
